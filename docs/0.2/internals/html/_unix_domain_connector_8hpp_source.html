38  /**
39  * \internal
40  *
41  * \brief Connect to a unix domain socket.
42  *
43  * This class connects to a unix domain socket and provides the following
44  * services:
45  * - Methods to connect and disconnect to the unix domain socket, and
46  * a method to obtain the current state,
47  * - A QT signal which is emitted when a PDU arrives (with an exception,
48  * see below),
49  * - A request service which sends a PDU and then blocks until the
50  * corresponding ResponsePDU arrived,
51  * - A send service which just sends a PDU.
52  *
53  * An object of this class is intended to run in its own thread, like so:
54  * \code
55  * QThread thread;
56  * UnixDomainConnector connection = new UnixDomainConnector();
57  * connection->moveToThread(&thread);
58  * thread.start();
59  * \endcode
60  * The UnixDomainConnector object communicates with its "outside world" via
61  * the QT signal/slot mechanism (which is thread-safe). In addition, the
62  * class offers methods which can directly be called from any thread and
63  * which are also thread-safe (and often use the signal/slot mechanism
64  * internally).
65  *
66  * \par Connection handling
67  *
68  * The UnixDomainConnector class uses QLocalSocket to communicate over a
69  * unix domain socket. It always starts in disconnected state, which means
70  * that the QLocalSocket is disconnected. The private slots do_connect()
71  * and do_disconnect() are invoked to connect resp. disconnect the socket.
72  * However, these slots should not be invoked from outside the object
73  * (therefore they are private slots). The methods connect() and
74  * disconnect() are offered to handle connection. These methods invoke
75  * do_connect() resp. do_disconnect().
76  *
77  * The connect() method blocks until the connection is established or a
78  * timeout is detected. It does so by invoking do_connect(), then waiting
79  * for a QWaitCondition to be triggered. The disconnect() method works the
80  * same way.
81  *
82  * The connection state is tracked with the m_is_connected member, which is
83  * protected by a mutex. The is_connected() method can access
84  * m_is_connected to inspect the state.
85  */
86  /**
87  * \par Sending and Receiving PDU's
88  *
89  * AgentX is a protocol based on a request-response model. A subagent can
90  * send a request and wait for a response, while the master can also send
91  * requests and expect a response. Furthermore, a subagent or a master can
92  * send multiple requests, then wait for all the responses. All those
93  * %PDU's can be interleaved.
94  *
95  * The UnixDomainConnector class handles sending and receiving PDU's
96  * separately. Sending is done using the do_send() slot, which works for
97  * all types of PDU: all request-PDU's (such as OpenPDU) can be send
98  * without considering special cases, and ResponsePDU's also are no
99  * exception. Received PDU's are forwarded using the pduArrived() signal to
100  * whoever is listening, which works for PDU's \e except ResponsePDU's:
101  * these are the answer to a sent request-PDU and must be routed
102  * differently.
103  *
104  * Received ResponsePDU's are transmitted via the m_responses map. This map
105  * assigns a packetID a ResponsePDU. Each time a request is sent, do_send()
106  * adds an entry to the map with the packetID of the request and a NULL
107  * ResponsePDU (i.e. a NULL pointer). This entry indicates that a
108  * ResponsePDU with the same packetID is awaited. The do_receive() slot
109  * then adds the ResponsePDU to the map, when it arrived. However, when a
110  * ResponsePDU arrives which is \e not awaited, it is discarded.
111  *
112  * \todo Improve error handling in all functions.
113  */

34 /**
35  * \brief Manage an SNMP table.
36  *
37  * This class manages a sequence of TableEntry objects. Unlike normal
38  * variables (which inherit from AbstractVariable), the table cannot be
39  * registered with a MasterProxy object. Instead, the table itself holds a
40  * pointer to a MasterProxy object and adds and removes its variables
41  * dynamically.
42  *
43  * To implement an SNMP table, a Table object is created and provided with
44  * - its own OID (which is the "root node" of all contained variables) and
45  * - a MasterProxy object.
46  * As long as the MasterProxy object is not provided, no entries can be added
47  * or removed from the table.
48  *
49  * Each TableEntry object has an index, which must be unique within a table.
50  * The Table class ensures this when adding a TableEntry.
51  *
52  * The OID's of the variables within a table are derived from the index of
53  * their TableEntry, which in turn is derived from the contents of the index
54  * variables of that very entry. The index variables are normal SNMP variables
55  * which can be changed (either programatically or by an SNMP set operation).
56  * In such a case, the TableEntry's index would change. However, the Table
57  * object is currently not able to detect such a change, i.e. variables will
58  * keep their OID instead of changing it according to the new index.
59  *
60  * \todo Table should detect index changes and re-add the variables
61  * accordingly. Suggestion: AbstractVariable should provide a
62  * valueChanged() signal, the Table object registers to this signal for
63  * all variables in an entry. If the signal is fired, the table checks
64  * whether the TableEntry index changed, and if so, re-adds the
65  * variables. I think QSignalMapper is needed.
66  */
